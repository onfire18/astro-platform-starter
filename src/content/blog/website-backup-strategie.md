Eine Website, die Jahre aufzubauen hat, kann durch einen Hackerangriff, einen Hosting-Fehler oder eine versehentliche Löschung innerhalb von Minuten verloren sein. Wer kein aktuelles Backup hat, steht dann vor einem teuren Neuaufbau – oder vor dem kompletten Verlust.

Backups sind die Versicherung Ihrer Website. Wie jede Versicherung: Man schätzt sie nicht, solange man sie nicht braucht.

## Was eine Website-Backup umfasst

Ein vollständiges Website-Backup besteht aus zwei Teilen:

**1. Datenbank.** Alle Inhalte – Texte, Seitenstruktur, Benutzerkonten, Einstellungen – sind in einer Datenbank gespeichert. Ohne Datenbankbackup verlieren Sie alle Inhalte.

**2. Dateisystem.** Alle Dateien auf dem Server – PHP-Code, Bilder, Uploads, Theme-Dateien, Plugin-Dateien. Ohne Dateisystem-Backup verlieren Sie Funktionalität und Medieninhalte.

Nur beides zusammen ist ein vollständiges Backup, aus dem die Website wiederhergestellt werden kann.

## Wie oft sollte gesichert werden?

Das hängt davon ab, wie oft sich Ihre Website ändert:

- **Täglich aktualisierte Websites** (Shops, Blogs, Portale): Tägliches Backup, mindestens wöchentliches Datenbank-Backup.
- **Gelegentlich geänderte Websites** (typische KMU-Website): Wöchentliches Backup ist oft ausreichend, ergänzt durch ein Backup nach jeder größeren Änderung.
- **Statische Websites**: Backup nach jeder Änderung plus monatliches Backup.

## Die 3-2-1-Backup-Regel

Ein bewährtes Prinzip aus der IT:

- **3 Kopien** Ihrer Daten (Original + 2 Backups)
- **2 verschiedene Speichermedien/Orte** (z.B. Server und Cloud)
- **1 Kopie außerhalb** des Primärstandorts (offline oder in einer anderen Cloud-Region)

Für Websites bedeutet das praktisch: Nicht nur das Hosting-Backup verlassen. Backups zusätzlich auf einem externen Dienst (z.B. Cloud-Speicher) oder einem lokalen Gerät speichern.

## Backups testen

Ein Backup, das nie getestet wurde, ist kein verlässliches Backup. Dateien können beschädigt sein, Prozesse können still scheitern, Datenbank-Dumps können inkomplett sein.

Testen Sie Ihre Backups regelmäßig: Stellen Sie eine Test-Instanz aus dem Backup wieder her und prüfen Sie, ob die Website funktioniert. Das muss nicht monatlich passieren, aber mindestens einmal pro Quartal ist sinnvoll.

## Wer ist für das Backup verantwortlich?

Das ist ein wichtiger Punkt, der in vielen Agentur-Kundenbeziehungen unklar ist:

**Hosting-Backups sind kein Ersatz.** Die meisten Hoster machen automatische Backups – aber diese dienen primär der Hoster-eigenen Infrastruktur, nicht dem Kunden-Schutz. Wie lange sie aufbewahrt werden, wie zuverlässig sie sind und ob Sie darauf zugreifen können, variiert stark.

**Klären Sie vorab, wer zuständig ist.** Wenn Sie mit einer Agentur oder einem Freelancer arbeiten: Wer macht die Backups? Wie oft? Wo werden sie gespeichert? Haben Sie darauf Zugriff? Das sollte vertraglich geregelt sein.

**Eigenverantwortung.** Letztlich liegt die Verantwortung für Ihre Daten bei Ihnen. Ein Backup, das nur beim Hoster liegt und das Sie nicht selbst herunterladen können, ist ein Sicherheitsnetz mit Löchern.

## Praktische Optionen für WordPress

WordPress-Websites können über Plugins automatisiert gesichert werden. Diese übertragen Backups in Cloud-Speicher (z.B. Google Drive, Dropbox, Amazon S3). Für nicht-WordPress-Websites gibt es serverseitige Lösungen oder Hosting-Pläne mit integriertem Backup-Service.

## Fazit

Ein Website-Backup ist kein technisches Detail für IT-Profis – es ist eine Grundvoraussetzung für den verantwortungsvollen Betrieb einer Website. Wer kein Backup hat, verlässt sich auf Glück. Wer ein regelmäßig getestetes Backup hat, kann im Ernstfall ruhig bleiben.
