Ein Website-Relaunch ist ein komplexes Projekt, bei dem viel schief gehen kann: SEO-Rankings einbrechen, Seiten nicht erreichbar sind, Formulare nicht funktionieren, Weiterleitungen fehlen. Eine systematische Checkliste verhindert die häufigsten Fehler.

Dieser Artikel ist eine praktische Übersicht – kein Ersatz für erfahrene Webentwickler bei komplexen Projekten.

## Vor dem Launch

### Inhalte und Design
- [ ] Alle Texte auf Richtigkeit und Vollständigkeit geprüft
- [ ] Alle Bilder optimiert (WebP-Format, sinnvolle Dateigröße)
- [ ] Alt-Texte für alle wichtigen Bilder vorhanden
- [ ] Impressum vollständig und korrekt
- [ ] Datenschutzerklärung aktuell und vollständig
- [ ] Cookie-Banner korrekt implementiert (Opt-in vor dem Setzen von Tracking-Cookies)
- [ ] Alle internen Links funktionieren
- [ ] Alle externen Links geprüft (keine toten Links)

### Technische Prüfungen
- [ ] HTTPS aktiv (SSL-Zertifikat installiert)
- [ ] Website auf verschiedenen Geräten getestet (Desktop, Tablet, Smartphone)
- [ ] Website in verschiedenen Browsern getestet (Chrome, Firefox, Safari, Edge)
- [ ] Alle Formulare getestet (Kontaktformular, Newsletter-Anmeldung)
- [ ] Formular-Bestätigungs-E-Mails kommen an
- [ ] PageSpeed-Score geprüft (Google PageSpeed Insights)
- [ ] Fehlerseite (404) vorhanden und ansprechend gestaltet
- [ ] Favicon vorhanden
- [ ] Meta-Titel und Meta-Beschreibungen für alle wichtigen Seiten

### SEO-Vorbereitung
- [ ] robots.txt korrekt konfiguriert (keine wichtigen Seiten ausgesperrt)
- [ ] XML-Sitemap vorhanden und aktuell
- [ ] Canonical-Tags gesetzt (falls nötig)
- [ ] Alte URLs gemappt: Für jede alte URL, die sich ändert, eine 301-Weiterleitung
- [ ] Google Search Console-Zugang vorhanden
- [ ] Google Analytics (oder Alternative) korrekt eingerichtet

### Weiterleitungen (besonders wichtig!)
Das größte SEO-Risiko beim Relaunch: Wenn alte URLs wegfallen, ohne Weiterleitungen zu setzen, verlieren Sie Rankings und bringen Besucher auf 404-Fehlerseiten.

- [ ] Liste aller bestehenden URLs der alten Website erstellt
- [ ] Für jede alte URL: Weiterleitung zur entsprechenden neuen URL eingerichtet
- [ ] Weiterleitungen getestet (301, nicht 302)

### Sicherung
- [ ] Vollständiges Backup der alten Website erstellt
- [ ] Datenbankbackup (falls vorhanden)
- [ ] Rollback-Plan definiert: Was tun, wenn etwas schiefgeht?

## Am Launch-Tag

- [ ] DNS-Änderungen durchführen (falls Domain wechselt oder Hosting wechselt)
- [ ] Propagierungszeit einplanen (24-48 Stunden für vollständige DNS-Propagierung)
- [ ] Direkt nach dem Launch: Alle wichtigen Seiten manuell aufrufen
- [ ] Formulare erneut testen
- [ ] Login-Bereich testen (falls vorhanden)
- [ ] E-Commerce-Checkout testen (falls vorhanden)
- [ ] Analytics-Tracking prüfen (kommen Daten an?)

## Nach dem Launch

### Erste Woche
- [ ] In Google Search Console: Sitemap neu einreichen
- [ ] In Google Search Console: Wichtige URLs zur Crawling-Anforderung eintragen
- [ ] Rankings für wichtige Keywords notieren (Ausgangspunkt für Vergleiche)
- [ ] Google Analytics: Überprüfen, ob Traffic-Daten korrekt erfasst werden
- [ ] Crawl-Fehler in Search Console prüfen

### Erster Monat
- [ ] Rankings beobachten: Gibt es starke Einbrüche? (Kann passieren – selten dauerhaft)
- [ ] Search Console: Welche Seiten werden gecrawlt? Fehler?
- [ ] Heatmap-Tool einrichten (Hotjar, Microsoft Clarity), um Nutzerverhalten zu beobachten
- [ ] Conversion-Tracking prüfen: Werden Anfragen/Käufe korrekt gemessen?

## Was Sie bei einem starken Ranking-Einbruch nach dem Launch tun sollten

Ein temporärer Ranking-Rückgang von 1-2 Wochen ist nach einem Relaunch normal – Google muss die neue Struktur erst crawlen und neu bewerten.

Wenn nach einem Monat die Rankings stark eingebrochen sind:
1. Search Console auf Crawl-Fehler prüfen
2. Weiterleitungen erneut prüfen (sind alle gesetzt? funktionieren sie?)
3. robots.txt prüfen (keine wichtigen Bereiche ausgesperrt?)
4. Duplicate Content prüfen (neue und alte URL beide zugänglich?)
